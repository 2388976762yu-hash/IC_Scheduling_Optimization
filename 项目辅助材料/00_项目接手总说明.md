## 1. 项目概述

### 1.1 项目名称
IC 后端制造生产排程优化仿真项目

### 1.2 项目背景
本项目围绕 **SanDisk 半导体后端制造场景**，基于已有订单数据、产品/物料数据和工艺约束，建立 Simio 离散事件仿真模型，用于模拟订单在后端制造各工序中的流转过程，并在此基础上研究生产排程优化问题。

项目当前重点不是做真实车间三维布局或搬运动画，而是先完成：

- 订单在关键工序中的逻辑流转
- 各工序加工时间估算
- 设备资源占用与 Worker 搬运
- 流转时间、利用率、Make Span、准交率等指标输出
- 后续支持排程策略、换型、故障等约束下的优化研究

### 1.3 核心研究目标
当前项目的核心目标是：

1. 基于现有订单与物料数据，建立可运行的 Simio 仿真模型（**当前为 Model 2.0**）；
2. 让模型能输出订单处理时间、设备利用率、Make Span、延迟惩罚等指标；
3. 逐步加入更贴近实际的约束，如：
   - 换型时间（DA/WB 等工序已部分接入）
   - 故障停机（Process 框架已存在，参数待校准）
   - 优先级/交期驱动派工
   - Worker 搬运与资源可用性
4. 最终支持以 **Make Span + 准交率** 为主目标，并兼顾利用率、WIP、流转时间等指标的排程优化研究。

## 7. 当前模型运行结果的关键发现（Model 2.0）

> 数据来源：`04_Output/v1v2_analysis.json` 与 V1/V2 报告（2026/5/22~5/23）

### 7.1 Source 放单问题：已在 Model 2.0 解决

Model 1.0 时期：41 单仅 13 单入流（28 balked）。

**Model 2.0 / V1/V2 结果**：

- `Source_Orders.OutputBuffer.Throughput = 41`
- `Order` 创建/销毁均为 41，完成率 **100%**
- 拆分后实体规模（V1 1×）：MLot 283、SubLot 652、Mag 1255

### 7.2 当前主要问题：负荷过低，利用率远低于目标

V1 原始场景（1×）平均利用率：

| 工序 | 平均利用率 | 设备数 |
|------|-----------|--------|
| DS | **13.7%** | 31 |
| DA | **13.5%** | 112 |
| WB | **13.4%** | 284 |
| Mold | ~8.9% | 7 |
| BG | ~9.0% | 17 |
| Taping | ~9.1% | 8 |

**结论**：

- 当前不是“DA/WB 瓶颈”，而是 **整体喂料不足 + 设备规模过大**
- 第七次会议亦指出：V1 仅约 10% 利用率，不符合开题/论文要求
- 需要通过 **订单扩容（×5/×6）、压缩 Release 窗口、调整有效设备数** 等方式做负荷校准

### 7.3 仿真时长与完成

- 当前实验多在 **10 天 / 7 天** 量级仿真设定下运行
- V2 场景 Sink 吞吐 628（SubLot 级完成统计），订单级 41 单全部完成
- 报告中有较高 **Off Shift** 占比（约 45%），解读利用率时必须考虑工作日历

### 7.4 已知 Trace 异常

`AUTO_Model2.0_Model_trace.csv` 中存在：

```
WB_Failed_1 → Seize1 → Referenced definition type 'OrderEntity' does not match executing type 'WB_Server'
```

说明 WB 故障 Process 的 Seize 节点配置可能有类型不匹配，需后续在 Simio 中排查。
