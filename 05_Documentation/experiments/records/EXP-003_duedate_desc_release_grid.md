## 与 EXP-002 对比

| 指标 | EXP-002 | EXP-003 | 变化 |
|------|---------|---------|------|
| Objective_min | **50.61** (1.0/0.75) | **52.38** (0.75/0.5) | 最优略**变差** |
| Penalty_max | 760 | **700** | 最差罚值略**降低** |
| 最优因子 | MLot=1.0, SubLot=0.75 | MLot=0.75, SubLot=0.5 | 偏移 |

**初判**：

1. DueDate **降序** 未全面改善结果；最优 Objective 略差于 EXP-002。
2. 高因子组合（1.25+）仍有显著 Penalty，说明释放顺序 alone 不足以消除延期。
3. 建议 **EXP-004**：改 DueDate **升序** + 同步检查 Source 是否按表序释放。

---

## 结论

- Experiment 已正常跑完（全 Scenario Completed）。
- Orders 降序调整带来**有限**改善（Penalty 峰值 760→700），整体最优 Objective 未优于 EXP-002。
- 下一步：DueDate **升序**释放 + Server 队列 EDD。
